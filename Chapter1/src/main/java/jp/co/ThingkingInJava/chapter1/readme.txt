Chapter1

对象的描述：对象具有状态，行为，标识
         - 状态：对象可以拥有内部数据(他们给出对象的状态)
         - 行为：方法(产生行为)
         - 标识：每一个对象都可以唯一的与其他对象区分开来(每个对象在内存中都有一个唯一的地址)。
接口：确定了特定对象所能发出的请求。

面向对象程序设计语言使用了 后期绑定的概念：当向对象发送信息时，被调用的代码直到运行时才会确定。
编译器确保被调用的方法的存在，并对调用参数和返回值执行类型进行检查(无法提供此类保证的语言为弱类型的)

一切都是对象
    用引用操作对象：java虽然一切都是对象，但是操作的标识符实际上是对象的一个引用(reference)。
    String s;这里创建的不是对象，而只是一个引用。此时向s发送一个信息，就会报错。此时s没有跟任何对象关联。

    java对象存储位置：注意不是JVM内存模型
        寄存器：最快的存储区，处理器内部储存
        堆栈：位于内存中，通过堆栈指针可以从处理器哪里获得直接支持。指针向下移动，开辟新的内存空间，向上移动，释放内存。
             创建程序时，系统必须知道存储在堆栈内所有项的确切生命周期，以方便移动指针。特别是对象的引用存在这里。

        非内存存储：以流或者持久化的方式。
